# Spec 002  /services (Projects)

## Purpose
Present projects as running API services.

## Content
- Each project = a service card: method badge (e.g. `POST /query`), name, one-line description, key tech, and a live demo / repo link.
- HONEST scoping. Example: Ask Your Corpus  describe as a contribution to a shared production app, owning the Postgres-native retrieval (eliminated separate vector DB, sub-second retrieval, BYOK). Do NOT imply sole ownership of the whole app.
- Show real, relevant detail (latency, tech) where truthful.

## Visual / motion
- Large editorial cards, generous gaps, one cyan accent.
- Scroll-triggered reveal with the shared easing.

## Acceptance
- Claims are accurate and defensible in an interview.
- Cards are scannable; links work.
