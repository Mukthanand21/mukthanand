# Spec 003  /changelog (Experience)

## Purpose
Journey and experience as a semver-style, dated changelog.

## Content
- Milestones as version bumps (e.g. `v1.4.0  built first RAG system`), each dated.
- Mix education, experience, and notable work. Newest first.
- Mono for version tags (violet, sparing).

## Visual / motion
- Clean vertical timeline, calm, lots of space.
- Entries reveal on scroll with shared easing.

## Acceptance
- Reads chronologically and clearly. No clutter.
